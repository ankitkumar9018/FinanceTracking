# FinanceTracker - Comprehensive Implementation Plan

## Vision
A highly visual, cross-platform personal investment tracking application for non-technical users investing in Indian and German markets. The app provides real-time portfolio monitoring, intelligent alerts, AI-powered insights, and beautiful data visualizations.

---

## Tech Stack

| Layer | Technology | Why |
|---|---|---|
| **Backend** | FastAPI 0.115+ (Python 3.12+) | Async-first, WebSocket native, auto OpenAPI docs |
| **Python Package Mgr** | **uv** (by Astral) | 10-100x faster than pip, lockfile support, replaces pip+venv+poetry |
| **Desktop App** | Tauri v2 + React 19 + TypeScript | Native feel, 5-10MB bundle vs Electron's 100MB+, Rust core |
| **Web App** | Next.js 15 + React 19 + TypeScript | App Router, RSC, streaming SSR |
| **Shared UI** | Shadcn/ui + Tailwind CSS 4 + Framer Motion | Beautiful, accessible, highly animated components |
| **Charts (Financial)** | TradingView Lightweight Charts v5 | 45KB, purpose-built for OHLC/candlestick, real-time |
| **Charts (Analytics)** | Apache ECharts 5 | Heatmaps, treemaps, portfolio pie/donut, rich tooltips |
| **Charts (Dashboards)** | Recharts + custom D3.js | Goal progress, risk gauges, sparklines |
| **Database ORM** | SQLAlchemy 2.0 async + Alembic | DB-agnostic (SQLite dev → PostgreSQL prod) |
| **Task Queue** | Celery + Redis | Background jobs: data fetching, ML training, alerts |
| **ML/DL** | PyTorch 2.x + scikit-learn + pandas_ta | Price prediction, anomaly detection, technical indicators |
| **LLM (Primary)** | **Ollama + Llama 3.2** (local) | Free, private, no API key needed, works offline |
| **LLM (Optional)** | OpenAI GPT-4 / Claude / Gemini via LangChain | Pluggable providers — system works fully without them |
| **Alerts** | ALL channels: Email (SendGrid) + WhatsApp (Twilio/Gupshup) + Telegram Bot + SMS (Twilio) + Desktop Push + In-App | Full multi-channel from day one |
| **Monorepo** | Turborepo + pnpm (JS) + uv workspaces (Python) | Fast builds, shared configs |
| **Testing** | pytest + vitest + Playwright | Unit, integration, E2E |
| **CI/CD** | GitHub Actions | Automated testing, building, releasing |

## Database Schema (Key Tables)

### Core Tables

**users** - id, email, password_hash, totp_secret, display_name, preferred_currency (INR/EUR), theme_preference, notification_preferences (JSON), created_at

**portfolios** - id, user_id, name, description, currency, is_default, created_at

**holdings** - id, portfolio_id, stock_symbol, stock_name, exchange (NSE/BSE/XETRA/etc), cumulative_quantity, average_price, lower_mid_range_1, lower_mid_range_2, upper_mid_range_1, upper_mid_range_2, base_level, top_level, current_price, current_rsi, action_needed (enum: N/Y_LOWER_MID/Y_UPPER_MID/Y_DARK_RED/Y_DARK_GREEN), custom_fields (JSON — user-defined columns), notes, sector, last_updated

**transactions** - id, holding_id, type (BUY/SELL), date, quantity, price, brokerage, notes, source (MANUAL/EXCEL/BROKER), created_at

**price_history** - id, stock_symbol, exchange, date, open, high, low, close, volume, rsi_14, created_at

**alerts** - id, user_id, holding_id, alert_type (PRICE_RANGE/RSI/CUSTOM), condition (JSON), is_active, last_triggered, channels (JSON: email/whatsapp/telegram/push)

**watchlist_items** - id, user_id, stock_symbol, exchange, target_buy_price, notes, lower_mid_range_1/2, upper_mid_range_1/2, base_level, top_level

### Extended Tables

**mutual_fund_holdings** - id, portfolio_id, scheme_code, scheme_name, folio_number, units, nav, invested_amount, current_value

**dividends** - id, holding_id, ex_date, payment_date, amount_per_share, total_amount, is_reinvested, reinvest_price, reinvest_shares

**tax_records** - id, user_id, transaction_id, financial_year, tax_jurisdiction (IN/DE), gain_type (STCG/LTCG/ABGELTUNGSSTEUER), purchase_date, sale_date, purchase_price, sale_price, gain_amount, tax_amount, holding_period_days

**goals** - id, user_id, name, target_amount, current_amount, target_date, category (RETIREMENT/HOUSE/EDUCATION/CUSTOM), linked_portfolio_id, monthly_sip_needed

**broker_connections** - id, user_id, broker_name, encrypted_api_key, encrypted_api_secret, access_token_encrypted, token_expiry, is_active, last_synced

**forex_rates** - id, from_currency, to_currency, rate, date, source

**chat_sessions** - id, user_id, messages (JSON), context (JSON), created_at

**notification_log** - id, user_id, channel, subject, body, status (SENT/FAILED/QUEUED), sent_at

**user_preferences** - id, user_id, column_order (JSON — ordered list of visible columns), custom_columns (JSON — definitions: name, type, default), table_density (compact/comfortable/spacious), default_chart_days (30), theme (dark/light/system)

**app_settings** - id, user_id, key (string, unique per user), value (encrypted string), category (string: notifications/brokers/ai/market_data/display/advanced), updated_at

## LLM Architecture: Graceful Degradation

The AI features are designed as **optional enhancements** — the core app (portfolio tracking, alerts, charts, tax) works 100% without any LLM:

```
┌─────────────────────────────────────────────┐
│           LLM Provider Registry             │
│                                             │
│  ┌─────────┐  ┌────────┐  ┌──────────┐     │
│  │ Ollama  │  │ OpenAI │  │  Claude   │     │
│  │ (Llama) │  │ (GPT4) │  │(Anthropic)│     │
│  │ PRIMARY │  │OPTIONAL│  │ OPTIONAL  │     │
│  └────┬────┘  └───┬────┘  └────┬─────┘     │
│       │           │            │            │
│       └───────────┼────────────┘            │
│                   ▼                         │
│        LLMService (abstract)                │
│        - chat(prompt) → response            │
│        - is_available() → bool              │
│        - fallback chain: Ollama→OpenAI→...  │
│                   │                         │
│                   ▼                         │
│        GracefulDegradation                  │
│        - If ALL providers fail:             │
│          → AI features show "AI offline"    │
│          → Core app continues normally      │
│          → No crashes, no errors            │
└─────────────────────────────────────────────┘
```

**Config in settings UI**: User picks provider (Ollama default), enters API keys for optional providers. Toggle AI features on/off.

## Visual Design Philosophy

The UI must be **highly visual** and **intuitive for non-technical users**:

### Core Visual Principles
1. **Dark & Light themes** with smooth transitions (Framer Motion) — dark theme as default for financial dashboards
2. **Color-coded everything**: Holdings table rows glow based on action needed
   - Light red pulse animation for lower mid range alerts
   - Light green pulse animation for upper mid range alerts
   - Dark red with warning icon for base level breach
   - Dark green with celebration icon for top level breach
3. **Sparkline mini-charts** inline in every table row showing 7-day trend
4. **Portfolio heatmap** (treemap visualization) showing allocation by size and performance by color
5. **Animated number transitions** when prices update in real-time (count-up/count-down effect)
6. **Interactive charts** - click any cell to expand into a full 30-day price/RSI chart overlay
7. **Dashboard cards** with gradient backgrounds showing total value, day P&L, top gainer/loser
8. **Goal progress** shown as animated radial gauges with confetti at milestones
9. **Risk meters** as semicircular gauges (speedometer style)
10. **Smooth page transitions** between sections (slide + fade)
11. **Skeleton loading states** for every data fetch (shimmer effect)
12. **Glassmorphism** card effects for the dashboard summary cards
13. **Responsive** - works beautifully on 4K monitors down to tablets

### Advanced Visual Features
14. **Live ticker bar** at top showing real-time price updates for top holdings (scrolling marquee)
15. **Candlestick charts with volume bars** — TradingView-quality financial charts
16. **Gradient P&L bars** — green gradient for profit, red gradient for loss, animated on data change
17. **Notification toast animations** — slide-in from top-right with stock icon and color
18. **Hover previews** — hover over any stock name to see a quick 7-day chart tooltip
19. **Drag-and-drop** Excel upload with file preview and animated progress ring
20. **Lottie animations** for empty states (no holdings yet, no alerts, etc.)
21. **Data density controls** — compact/comfortable/spacious table views
22. **Mini portfolio widget** — small floating widget showing total value + day change (always visible)

## Live Data Strategy: How We Get Real-Time Data

Ensuring all required live data is actually obtainable:

### Current Stock Price

| Source | Type | Coverage | Cost | Latency | Used When |
|---|---|---|---|---|---|
| **Broker WebSocket** (Kite/Breeze/SmartAPI) | Real-time streaming | NSE/BSE | Part of broker API | < 1 second | Broker connected |
| **yfinance** | Polling (REST) | NSE/BSE/XETRA/global | Free | 15-min delayed (NSE) | No broker / fallback |
| **Twelve Data API** | Polling (REST) | NSE/BSE + 50 exchanges | Free tier: 800 calls/day | 1-min delayed | Optional upgrade for faster data |

**Strategy**: If broker connected → use broker WebSocket (real-time). Otherwise → yfinance every 5 min. If yfinance fails → show last cached price with timestamp.

**Indian stocks**: Append `.NS` (NSE) or `.BO` (BSE) suffix for yfinance. Example: `"RELIANCE.NS"`, `"TCS.NS"`.
**German stocks**: Append `.DE` for XETRA. Example: `"SAP.DE"`, `"SIE.DE"`.

### RSI (Relative Strength Index)

| Method | How | Accuracy | Used When |
|---|---|---|---|
| **Self-calculated** via `pandas_ta` | Fetch 30+ days of closing prices → calculate RSI-14 | Exact (standard formula) | Always (primary) |
| **Broker API** | Some brokers provide pre-calculated technical indicators | Exact | If broker offers it |
| **Twelve Data API** | `/technical_indicators/rsi` endpoint | Exact | Optional enhancement |

**Strategy**: Always self-calculate RSI from historical close prices using `pandas_ta.rsi(close, length=14)`. This is the most reliable method and doesn't depend on any external API providing RSI directly. Updated daily after market close + on-demand recalculation.

### Historical OHLCV Data (for charts)

| Source | Max History | Intervals | Coverage |
|---|---|---|---|
| **yfinance** | 20+ years | 1d, 1wk, 1mo, 1h (last 730 days), 5m (last 60 days) | Global |
| **Zerodha Kite** | Per API tier | 1min, 5min, 15min, 30min, 1h, 1d | NSE/BSE |
| **ICICI Breeze** | 10 years | 1-second OHLCV | NSE/BSE |
| **Twelve Data** | Varies by plan | 1min to 1month | Global |

**Strategy**: Use yfinance for historical data (free, reliable, 20+ years). Cache in `price_history` table. Broker data supplements with higher resolution (intraday) when connected.

### Technical Indicators (beyond RSI)

All calculated locally using `pandas_ta` from cached OHLCV data:
- **RSI-14**: `ta.rsi(close, length=14)`
- **MACD**: `ta.macd(close, fast=12, slow=26, signal=9)`
- **Bollinger Bands**: `ta.bbands(close, length=20, std=2)`
- **SMA/EMA**: `ta.sma(close, length=20)`, `ta.ema(close, length=50)`
- **Support/Resistance**: Calculated from swing highs/lows using custom algo
- **Fibonacci**: Calculated from recent swing high to swing low

No external API needed — all computed from stored price data.

### Forex Rates (INR/EUR)

| Source | Cost | Update Frequency |
|---|---|---|
| **exchangerate-api.com** | Free: 1500 calls/month | Daily |
| **ECB (European Central Bank)** | Free | Daily (published ~16:00 CET) |
| **yfinance** | Free | `"EURINR=X"` real-time |

**Strategy**: Fetch INR/EUR rate daily from ECB (official, free, reliable). Supplement with yfinance for intraday. Cache in `forex_rates` table.

### News & Sentiment Data

| Source | Type | Coverage |
|---|---|---|
| **RSS Feeds** | Free | MoneyControl, Economic Times, LiveMint, Reuters India |
| **Google News RSS** | Free | Global + India-specific queries |
| **NewsAPI.org** | Free tier: 100 calls/day | Global headlines |

**Strategy**: Fetch news via RSS feeds (free, no API key). Sentiment scored locally using FinBERT model (runs on CPU, no GPU needed). Updated every 30 minutes during market hours.

### Mutual Fund NAV

| Source | Cost | Coverage |
|---|---|---|
| **MFapi.in** | Free | All Indian MFs, daily NAV |
| **AMFI website** | Free | Official NAV data |
| **mftool** (Python) | Free | Python wrapper for AMFI |

**Strategy**: Use `mftool` library → fetches from AMFI directly. Updated daily after NAV published (~11 PM IST).

## Graceful Degradation: Everything Must Work

The app is designed so **no single dependency failure breaks the system**:

| Service | If Missing/Down | Fallback Behavior |
|---|---|---|
| **Redis** | Not installed or down | Use in-memory task queue (`huey` with SQLite backend or `asyncio` background tasks). Alert checking runs in-process on a timer instead of Celery. |
| **Celery** | Not installed | Background tasks run as `asyncio` scheduled tasks within the FastAPI process. Slower but functional. |
| **Ollama / LLM** | Not installed or all providers offline | AI chat shows "AI assistant offline" banner. All non-AI features work normally. AI insights card hidden. |
| **yfinance** | API rate limited or down | Shows "Price data temporarily unavailable" with last known price + timestamp. Retries with exponential backoff. |
| **Broker API** | Connection fails | Manual data entry still works. Shows "Broker disconnected" badge. Cached last-sync data displayed. |
| **SendGrid / Email** | Not configured | Email alerts disabled. Other channels (Telegram, WhatsApp, in-app, desktop push) still work. |
| **Twilio / WhatsApp** | Not configured | WhatsApp/SMS alerts disabled. Other channels work. |
| **Telegram Bot** | Not configured | Telegram alerts disabled. Other channels work. |
| **Internet** | Offline | Desktop app shows cached portfolio data with "Offline" indicator. All data entry works. Syncs when back online. |
| **PostgreSQL** | Not available | Falls back to SQLite (default). Config in `.env`. |

**Implementation pattern**: Every external service call is wrapped in:
```python
async def safe_call(service_fn, fallback_fn, *args):
    try:
        return await service_fn(*args)
    except (ConnectionError, TimeoutError, ServiceUnavailable):
        logger.warning(f"{service_fn.__name__} unavailable, using fallback")
        return await fallback_fn(*args)
```

**Startup health check**: On app launch, check all services and show status dashboard:
```
Service Status:
  Database:  SQLite ✅
  Redis:     Not found ⚠️ (using in-memory queue)
  Ollama:    Running ✅ (llama3.2)
  yfinance:  Connected ✅
  Email:     Not configured ⚠️
  WhatsApp:  Not configured ⚠️
  Telegram:  Connected ✅
```

## Startup & Setup Script

A single script `start.sh` (macOS/Linux) and `start.ps1` (Windows) that handles everything:

### What the script does:
```bash
#!/bin/bash
# start.sh - FinanceTracker Setup & Launch Script

echo "=== FinanceTracker Setup & Launch ==="

# 1. CHECK PREREQUISITES
echo "Checking prerequisites..."
check_and_install "python3" "3.12"     # Check Python 3.12+
check_and_install "node" "20"          # Check Node.js 20+
check_and_install "pnpm"               # Check pnpm
check_and_install "uv"                 # Check uv (Python package manager)
check_optional "redis-server"          # Check Redis (optional)
check_optional "ollama"                # Check Ollama (optional)

# 2. INSTALL DEPENDENCIES (if not already installed)
echo "Installing backend dependencies..."
cd backend && uv sync                  # Install Python deps from uv.lock
cd ..

echo "Installing frontend dependencies..."
pnpm install                           # Install all JS deps

# 3. SETUP DATABASE
echo "Running database migrations..."
cd backend && uv run alembic upgrade head
cd ..

# 4. STOP EXISTING SERVICES (if running)
echo "Stopping existing services..."
kill_if_running "uvicorn"              # Stop backend
kill_if_running "next-server"          # Stop web app
kill_if_running "celery"               # Stop Celery worker
# Note: Redis and Ollama are not stopped (shared services)

# 5. START SERVICES
echo "Starting services..."

# Start Redis (if available)
if command_exists redis-server; then
    redis-server --daemonize yes
    echo "  Redis: ✅ Started"
else
    echo "  Redis: ⚠️ Not found (using in-memory fallback)"
fi

# Start Ollama (if available)
if command_exists ollama; then
    ollama serve &
    echo "  Ollama: ✅ Started"
else
    echo "  Ollama: ⚠️ Not found (AI features disabled)"
fi

# Start backend
cd backend && uv run uvicorn app.main:app --reload --port 8000 &
echo "  Backend: ✅ http://localhost:8000"

# Start Celery worker (if Redis available)
if redis_running; then
    cd backend && uv run celery -A app.tasks.celery_app worker --beat -l info &
    echo "  Celery: ✅ Started"
fi

# Start web app
cd apps/web && pnpm dev &
echo "  Web App: ✅ http://localhost:3000"

# 6. HEALTH CHECK
sleep 5
echo ""
echo "=== Service Status ==="
check_health "Backend"  "http://localhost:8000/health"
check_health "Web App"  "http://localhost:3000"
check_health "Redis"    "redis://localhost:6379"
check_health "Ollama"   "http://localhost:11434"

echo ""
echo "🎉 FinanceTracker is running!"
echo "   Web:     http://localhost:3000"
echo "   API:     http://localhost:8000"
echo "   Docs:    http://localhost:8000/docs"
```

  - Price NOT in any mid range → action_needed = "N"
  - Price in lower mid range (between lower_mid_range_2 and lower_mid_range_1) → "Y" + LIGHT_RED
  - Price in upper mid range (between upper_mid_range_1 and upper_mid_range_2) → "Y" + LIGHT_GREEN
  - Price between lower_mid_range_1 and base_level (or below base) → "Y" + DARK_RED
  - Price between upper_mid_range_2 and top_level (or above top) → "Y" + DARK_GREEN
- Runs every minute during market hours via Celery beat

## Security Architecture

1. **Authentication**: JWT (access: 15min, refresh: 7d) + optional TOTP 2FA
2. **Encryption at rest**: Broker API keys encrypted with Fernet (symmetric key from env)
3. **Encryption in transit**: HTTPS only, HSTS headers
4. **CORS**: Strict origin allowlist
5. **Rate limiting**: slowapi middleware (100 req/min general, 10 req/min for auth)
6. **Input validation**: Pydantic models validate all inputs
7. **SQL injection**: SQLAlchemy ORM parameterized queries (no raw SQL)
8. **XSS**: React auto-escapes, CSP headers
9. **Secrets management**: Environment variables via `.env` (never committed)
10. **Audit log**: All sensitive actions logged with timestamp and user ID
